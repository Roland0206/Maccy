# Archive privacy defaults

Archive storage keeps clipboard data on disk for longer than popup history. Defaults favour bounded non-text retention:

- Plain text: Forever
- Rich text & web content: 30 days
- Images: 7 days
- Files: 30 days
- Other / unknown binary: 7 days

Maccy never archives items ignored by existing privacy controls:

- `org.nspasteboard.TransientType`
- `org.nspasteboard.ConcealedType`
- `org.nspasteboard.AutoGeneratedType`
- user ignored pasteboard types
- default or user ignored source apps
- ignored text regex matches

Default ignored source apps include common password managers: 1Password, Bitwarden, Dashlane, LastPass, KeePass/KeePassXC, Enpass, and Strongbox. Users can remove or extend this list in Ignore settings.

Image OCR is off by default. Users must enable **Storage → Recognize text in images** before Maccy runs on-device OCR for image titles/search text.

## Delete behavior

Archive deletion uses tombstones first, then later physical cleanup. This prevents deleted rows from reappearing during maintenance/import flows.

Secure delete is not guaranteed. SQLite WAL files, filesystem snapshots, SSD wear leveling, backups, and external payload files may retain recoverable bytes after normal deletion until OS/storage cleanup removes them. For high-sensitivity workflows, keep source apps or pasteboard types ignored instead of relying on deletion after capture.
